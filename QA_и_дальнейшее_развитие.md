# QA-чек-лист и дальнейшее развитие бота ОБНУЛЕННЫЙ

Этот файл — на случай, если проверку и доработку продолжит другая сессия (например, Claude Code с реальным доступом в интернет). Здесь то, что я не смог протестировать вживую из песочницы без доступа к api.telegram.org и внешним API, плюс идеи для роста.

## Что реализовано в этой сессии

- Инлайн-меню и навигация кнопками вместо ручного ввода команд (`Keyboards`, `on_callback`)
- Единый HTML-визуальный стиль всех карточек
- Технический анализ `/ta` с графиком (matplotlib, тёмная тема, SMA20 + Bollinger Bands)
- Мультибиржевые цены `/exchanges` через ccxt (Binance/Bybit/Kraken/OKX)
- Скринер `/screener`, тренды `/trending` (CoinGecko `/search/trending`)
- Новости `/news` через RSS CoinDesk + Cointelegraph (замена сломавшегося CryptoCompare API, который теперь требует ключ)
- **Автопостинг новостей в группу**: JobQueue проверяет RSS каждые 20 минут, публикует новые новости (дедуп по URL), автотегирует монету в заголовке и добавляет кнопки «ТА»/«Анализ»/«Читать»
- Авто-определение группы сообщества (по первому сообщению/входу участника) + команды `/setgroup`, `/autopost on|off` (только для админов группы)
- Лимиты free-тира на премиум-фичи, апсейл на Standard
- Исправлен баг с `callback_data` >64 байт для длинных coin_id (через индирекцию `context.user_data['cur_coin']`)

Все перечисленное прогнано через юнит-тесты на моках (сеть подменена), синтаксис проверен `py_compile`, зависимости проверены в чистом venv (`pip check` — без конфликтов).

## Что нужно проверить вживую (не смог протестировать из песочницы)

1. **JobQueue реально стартует и стреляет по расписанию на Railway.** Логика проверена изолированно (вызов job-функции напрямую), но сам `run_repeating` через APScheduler в реальном процессе — нет. Смотри логи Railway на строку с публикацией новости или ошибки APScheduler при старте.
2. **RSS-ленты отдают ожидаемый формат.** Confirmed доступность доменов (coindesk.com/arc/outboundfeeds/rss, cointelegraph.com/rss) через веб-поиск и один fetch, но реальный XML не разбирал — если формат отличается от ожидаемого `<channel><item><title>/<link>/<pubDate>`, парсинг может возвращать пустой список. Проверить: `/news bitcoin` в боте.
3. **Определение группы (`_maybe_register_group`).** Сработает при первом сообщении в группе после деплоя. Если бот уже давно в группе и участники не пишут — стоит один раз явно вызвать `/setgroup` от админа.
4. **ccxt против реальных бирж.** Публичные тикеры Binance/Bybit/Kraken/OKX меняют форматы символов (например, Kraken не всегда торгует парой XXX/USDT). Код это обрабатывает (пропускает биржу с ошибкой), но стоит прогнать `/exchanges BTC`, `/exchanges SOL` и посмотреть, сколько бирж реально отвечает.
5. **Rate limits CoinGecko публичного API** (~10-30 запросов/мин). `/screener` и `/trending` дергают его чаще остальных — при активном использовании сообществом может потребоваться либо платный CoinGecko API-ключ, либо кэширование (см. ниже).

## Идеи для дальнейшего развития

- **Кэширование данных CoinGecko** (например, 60 сек TTL в памяти) — снизит риск rate-limit при росте аудитории
- **Миграция community_data.json → SQLite/Postgres** — JSON-файл на диске не потокобезопасен при параллельных апдейтах и не переживёт передеплой на некоторых хостингах без persistent volume
- **Telegram Stars** — реальная оплата Standard/Pro тарифов прямо в боте (сейчас только текст «скоро»)
- **Ежедневный дайджест** — отдельная джоба (утро по будням), постящая топ-движения рынка в группу для Standard/Pro
- **Сентимент-теги у новостей** — простая LLM-классификация заголовка (позитив/негатив/нейтрально) вместо голого списка
- **Админ-панель** — веб или отдельные команды для просмотра метрик сообщества (DAU, конверсия Free→Standard, retention)

## Быстрый чек-лист после деплоя

```
/start          → открывается меню с кнопками
/ta bitcoin     → приходит график + текст
/exchanges BTC  → минимум 2-3 биржи отвечают ценой
/news bitcoin   → есть хотя бы 1 новость
/trending       → список монет без ошибки
(в группе) дождаться автопоста новости или проверить /autopost → «включён»
```
